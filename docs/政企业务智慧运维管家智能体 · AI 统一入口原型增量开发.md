## 一、背景与任务性质

**本次要做的**：在**现有 React Web 原型基础上**，**新增**一套"AI 统一入口"，把原本分散在多个页面的能力（知识库、自助排障、智能问答、运行报告等）聚合到一个**悬浮式 AI 入口浮层**里。

**关键原则**：
- **不要重写现有页面**，不要改动 `pages/agent/` 之外的任何文件
- 尽可能**不要引入现有项目没有的新库**（UI 库、状态管理、图表、动效等优先复用）
- **优先复用现有组件和 design tokens**，视觉风格与现有系统保持一致
- 现有页面不受影响，客户可以继续用原路径访问原页面；AI 入口是叠加上去的新能力，通过新的路访问

## 二、角色与受众

原型涉及三方角色，务必区分：

- **方案作者（我）**：厂商产品经理，为运营商建设该系统
- **原型观看者**：运营商业务线负责人（汇报对象）
- **原型模拟的使用者**：运营商侧的**政企最终用户**（安徽地区的政企企业业务 / IT 负责人，非专业网络人员）

**产品定位一句话**：让不懂网络的政企最终用户，像看天气预报一样了解自己的业务、像叫外卖一样报障、像查快递一样跟进处理。

## 三、技术栈与工程约束

- **技术栈**：**React Web 端**（**不是移动 APP，不要套移动端容器、不要用移动端手势**）
- **所有具体选型**（React 版本、UI 组件库、状态管理、路由、样式方案、图表库、动效库等）**严格复用现有项目已有的依赖**，不要引入新库
- **目录约定**：所有新增代码统一放在 **`pages/agent/`** 下（如目录已存在则在其中新建 `ai-dock/` 子目录承载本次新增内容，具体由你判断，不要与现有文件冲突）
- **样式**：直接复用现有项目的 design tokens 和样式变量文件，色彩 / 字号 / 间距 / 圆角 / 阴影全部走现有规范
- **通用组件优先复用**：现有项目已封装的按钮、卡片、Modal、Toast、Tabs、Drawer、Progress 等组件**直接 import 使用**，不要自己重复实现
- **数据**：全部使用**本地 mock 数据**（TS 常量或 JSON），不调用任何真实后端
- **响应式**：桌面端优先（1440×900 及以上），**按 Web 端设计**（鼠标 hover、右键、键盘快捷键都可用），不要求适配移动端
- **交互保真度**：所有按钮必须有响应（真实跳转或 toast 模拟），不允许有"死按钮"

## 六、统一 AI 入口 · 产品形态（核心架构约束）

> **这一章是最高优先级约束，与后文任何内容冲突时以本章为准。**

### 6.1 现状与目标

**现状**：现有原型中已经有分散的知识库、自助排障、智能问答、运行报告等独立页面（由现有页面承载，不动）。

**目标**：新增一个**统一 AI 入口**浮层，把这些能力**再聚合一次**，让用户可以在一个对话 + 面板混合界面中完成所有操作，不必在多个独立页面之间跳来跳去。

**共存原则**：
- 现有独立页面**保留**，原路径继续可用
- AI 入口是**叠加**的新形态，用户可以二选一
- 演示时重点展示 AI 入口，体现"统一收口"的设计升级

### 6.2 入口形态

**形态**：Web 页面右下角常驻一个**悬浮球** → 点击后展开为**浮层面板**（Web 端右下角浮起，不跳转新页面、不打开新窗口）。浮层内部采用**"对话主场 + 能力 Tab"**的混合结构：

```
┌─────────────────────────────────────────────┐
│  管家 · AI 入口                      [_][□][x]│  顶部栏（最小化/最大化/关闭）
├─────────────────────────────────────────────┤
│  [💬对话] [📊业务概览] [🔧自助排障]          │  能力 Tab（可横向滚）
│  [📄服务报告] [📚知识库] [📋工单进度]         │
├─────────────────────────────────────────────┤
│                                             │
│  （主内容区：根据当前 Tab 切换显示）          │
│                                             │
├─────────────────────────────────────────────┤
│  [输入框]                              [发送] │  底部对话输入（常驻）
└─────────────────────────────────────────────┘
```

### 6.3 三条核心设计原则

**原则 1 · 对话是主场，Tab 是快捷入口**
- 对话 Tab 永远是默认落地态
- 切到其他 Tab 时展示的不是"完整页面"而是**结构化面板**
- 任一 Tab 操作完成后，结果**自动回流到对话 Tab**，形成一条对话消息
- 例：自助排障 Tab 点"一键体检" → 诊断报告生成 → 对话区同步新增管家消息

**原则 2 · 对话可调用所有能力**
- 用户说"帮我看一下专线" → 对话里直接返回业务概览卡片
- 用户说"我要报障" → 对话里**原地**展开报障表单（不跳 Tab）
- 用户说"我看看月报" → 对话里返回月报摘要卡片
- **Tab 只是可视化捷径，不做任何对话做不到的事**

**原则 3 · 能力卡片可嵌入对话消息**

对话消息支持以下富媒体类型（均为 React 组件，通过消息对象 `type` 字段路由渲染）：

| 消息类型 | 使用场景 |
|------|------|
| `TextMessage` | 文字回答（结论 + 解释 + 建议结构） |
| `BusinessCardMessage` | 业务状态卡片，点击原地展开详情抽屉 |
| `DiagnosisCardMessage` | 诊断报告卡片（折叠态 → 展开态） |
| `TicketCardMessage` | 工单进度卡片（带进度条） |
| `ReportCardMessage` | 月报 / 周报摘要卡片 |
| `FormMessage` | 表单嵌入对话（主要用于报障） |
| `ActionButtonsMessage` | 推荐动作按钮组 |
| `LoadingMessage` | 加载中状态 |

### 6.4 底层页面

AI 入口是**悬浮在现有页面之上**的。用户访问原型时看到的是现有 React 原型的正常页面，右下角多了一个悬浮球。**不要专门做一个"底层背景"**——现有页面就是底层。

---

## 七、悬浮球设计

- 常驻当前页面**右下角**，`position: fixed`
- 有消息 / 告警时显示**红点数字**
- **鼠标 hover** 时弹出气泡预览（最近一条重要消息 + 健康度摘要）
- **点击**展开浮层面板
- 演示控制台放在**左下角**（独立浮层），不要和 AI 入口混在一起

---

## 八、浮层面板的尺寸与交互

- **默认尺寸**：右下角浮起，宽 480px、高 720px
- **最大化态**：点击顶部栏方框图标 → 铺满屏幕 80%，用于看报告 / 工单详情等长内容
- **最小化态**：收起为悬浮球原状，红点标记未读
- **拖拽**：顶部栏可拖拽移动（Web 端鼠标拖拽）
- **键盘快捷键**：`Esc` 关闭、`Cmd/Ctrl + K` 聚焦输入框、`Cmd/Ctrl + /` 切换最大化（可选实现）
- **遮罩策略**：最大化态下半透明遮罩覆盖底层页面；默认态不遮罩（用户可以一边用管家一边看原页面）

## 九、6 个 Tab 的详细设计

### 9.1 💬 对话 Tab（默认）

- **顶部**：欢迎语 + 当前业务健康度摘要（一句话 + 小健康度环形图）
  - 欢迎语示例："您好，合肥 XX 集团，当前业务整体良好"
- **推荐问题区**：4–6 个按钮，根据当前状态动态变化
  - 全绿时：推荐"这个月我的业务用得怎么样""量子加密保护是什么"
  - 有告警时：推荐"刚才为什么报警""上次类似问题怎么解决的"
- **消息流区**：按时间顺序渲染消息气泡，支持富卡片消息
- **底部输入框**：支持文字输入，左侧"+"按钮可快速插入常用模板
- **管家主动消息**：演示控制台触发后，管家主动推消息到对话流

### 9.2 📊 业务概览 Tab

**上半部分 · 整体健康度**
- 大环形健康度指示器（绿 / 黄 / 红三态）
- 一句话结论：如"当前一切正常 ✅" / "有 1 项需要关注 ⚠️" / "有 1 项故障需处理 🔴"

**顶部通知条 · 我的订单（售中入口）**
- 小条带样式，例："您有 1 个 5G 专网开通中，预计 3 天后可用。[查看]"
- 点击展开订单抽屉（进度条 + 每步预计完成时间 + 当前卡点 + 简单变更入口）

**中部 · 五类业务状态卡片**
每张卡片：
- 状态灯（异常态用呼吸动效，不要闪烁）
- 业务名称 + 数量（如"专线 · 6 条"）
- 1–2 个关键指标（翻译后语言）
- 异常卡片自动置顶

**业务详情 · 点击卡片原地展开抽屉**

| 业务 | 抽屉核心内容（数据本地化到安徽） |
|------|------|
| **专线** | 节点图（如"合肥总部 ↔ 阜阳"通断状态）+ 带宽使用率 + 本月中断次数 + 最近 7/30 天趋势 |
| **5G 专网** | 覆盖区域（如"合肥高新区某厂"）+ 在线终端数 + 本月流量 + 异常终端 |
| **IDC 动环** | 机房温度 / 湿度 / 供电 / 门禁四个方块 + 历史曲线（如"合肥滨湖机房"） |
| **量子+SD-WAN** | 安徽省内分支拓扑（16 地市示意）+ **量子加密保护状态**醒目标识 |
| **智算** | 算力使用率 + 任务排队数 + 本月费用 + 任务列表 |

> 拓扑 / 关系图复用现有项目的图表库，无现成封装则用 SVG 简单示意，不要引入新库。

所有抽屉通用：
- 顶部状态横幅（一句话结论）
- 关键指标（翻译后语言 + 术语小问号图标 hover 解释）
- 趋势图
- 异常事件列表
- 底部操作：[一键体检] [查看历史] [报障] [咨询客户经理]

### 9.3 🔧 自助排障 Tab

- **业务选择器**：下拉或卡片式选择五类业务之一
- **一键体检按钮**：大按钮，点击后 loading 3–5 秒，生成诊断报告
- **报障入口**：快捷按钮，直接进入报障表单
- **历史诊断记录**：列表形式

**诊断报告结构**（严格遵守）：

```
🔴 问题判断
  一句话结论（例：合肥总部到阜阳专线当前中断，已持续 12 分钟）

👥 影响范围
  · 阜阳分公司 78 名员工
  · 受影响业务：ERP、OA、视频会议、文件共享
  · 严重程度：高

🔍 可能原因（按可能性排序，3 条）
  1. 阜阳侧接入设备告警（可能性 70%）
  2. 光缆中段异常（可能性 25%）
  3. 配置变更影响（可能性 5%)

👉 建议操作
  [一键报障]   [查看历史类似故障]   [联系客户经理]

ℹ️ 数据来源：本次诊断基于最近 15 分钟的运行数据、告警信息
```

- 诊断报告同时**回流到对话 Tab**（核心机制）
- 支持"复制 / 导出为 PDF"（模拟下载，toast "已下载"）

**报障表单（从诊断报告"一键报障"进入）**
- 所有字段自动预填（业务信息、故障现象、诊断结果、联系人、相关历史工单）
- 用户补充 0–1 句备注即可
- 提交后对话 Tab 显示"工单已创建"消息 + 工单进度卡片

**报障表单（从"自助排障" Tab 直接进入）**
- 手动选择业务
- 现象支持标签快选 + 文本补充
- 提交后系统自动诊断并附到工单

### 9.4 📄 服务报告 Tab

**顶部 tab 切换**：周报 / 月报 / 季报

**报告卡片列表**：按时间倒序

**封面内容**：
- 一句话总结（例："本月总体良好，1 次小故障已恢复"）
- 健康度星级（⭐⭐⭐⭐☆）
- 时间范围

**点击卡片 → 浮层自动最大化展示报告全文**：
- 本月亮点
- 需关注事项
- 各业务数据速览
- 合同承诺达标情况（进度条，**不能出现"SLA"字样**）
- 异常事件清单
- 趋势对比（同比 / 环比）
- 优化建议

**右上角操作**：
- [导出 PDF]、[导出为长图]（均模拟下载）
- [发送给同事]（弹出联系人选择弹窗）

### 9.5 📚 知识库 Tab

- 左侧分类导航（按五类业务 + 通用类别）
- 每分类下展示 5–10 个高频问题
- 顶部搜索框
- **点击问题 → 回到"💬 对话" Tab，并在对话流展示该问题的标准回答**（结论 + 解释 + 建议结构）

### 9.6 📋 工单进度 Tab

**工单列表**：按状态分组（处理中 / 已恢复 / 已关闭），每条显示工单号、业务、简短描述、状态徽标、更新时间

**点击工单展开详情（同 Tab 内）**：
- 顶部：工单编号 + 状态徽标
- 中部：进度条（已受理 → 派单中 → 处理中 → 已恢复 → 待回访）
- 右侧：当前处理人、预计恢复时间、最新动作（带时间戳）
- 底部按钮：[催办] [补充说明] [申请升级] [查看更多]

**演示交互**：
- 催办 → toast "已催办"
- 申请升级 → 确认弹窗
- 补充说明 → 简单输入框
- **工单状态变更通过演示控制台推进**，每次推进对话 Tab 自动生成主动消息

## 十、演示控制台（左下角独立浮层）

Web 端现场演示用的彩蛋面板，开关包括：

## 十一、交互与动效

## 十四、演示场景脚本（4 条主线，必须全部跑通）

全部**在 AI 入口浮层内完成，不跳页面**：

### 场景 A · 看一眼就知道没事（30 秒）

访问 Web 原型 → 点右下角悬浮球
→ 浮层展开，默认"💬 对话" Tab，顶部"当前一切正常 ✅"
→ 点推荐问题"给我看看所有业务状态"
→ 对话里返回五张业务卡片消息（专线 6 条 / 5G 专网 2 张 / IDC 3 个 / 量子SD-WAN 1 张 / 智算）
→ 点专线卡片 → 原地展开详情抽屉（合肥总部到各地市分支状态）
→ 关闭抽屉 → 最小化浮层

### 场景 B · 从异常到解决（不跳页面，2 分钟）

点悬浮球 → 管家主动推告警："您好，合肥总部到阜阳专线 3 分钟前出现告警 ⚠️"（附业务卡片 + 推荐动作）
→ 点"立即体检" → loading"正在为您体检..."（2–3 秒）
→ loading 变为诊断报告卡片（折叠态）
→ 点展开 → 完整诊断报告
→ 点"一键报障" → 对话流**原地**出现预填表单
→ 确认提交 → "工单已创建"消息 + 工单进度卡片
→ 演示控制台推进工单（派单 → 处理中 → 已恢复）
→ 每次推进对话流收到主动消息

### 场景 C · 看月报（30 秒）

点悬浮球 → 切"📄 服务报告" Tab → 选月报 → 点某份月报
→ 浮层自动最大化展示报告全文
→ 翻到"合同承诺达标情况"段
→ 点"导出 PDF" → toast "已下载"
→ 退出最大化

### 场景 D · 问问题 + 降级演示（1 分钟）

点悬浮球 → 在对话 Tab 点推荐问题"阜阳到合肥总部专线今天正常吗？"
→ 返回结构化回答（💡结论 + 📖解释 + 👉建议）+ 业务卡片
→ 点"查看专线详情" → 原地展开抽屉 → 关闭
→ 演示控制台切到"纯 FAQ 模式"
→ 输入识别不了的问题"请帮我分析一下下季度业务规划"
→ 对话展示兜底卡片："我暂时回答不了，但可以帮您转客户经理"（附一键转人工按钮）
