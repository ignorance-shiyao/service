## 一、布局总览

采用4行×4列共16格布局(沿用现有管家形态)，地图模块横跨2格。实际占位15个模块。部分模块内置Tab切换，在同一格内承载相近维度的信息。

```
┌──────────────────────────────────────────────────────────────────────────┐
│  政企业务智慧运维管家 - 量子+SD-WAN管家 - XX客户                            │
│  [业务总览] [性能可视] [资源可视] [告警可视] [服务台] [消息通知] [系统管理]     │
├─────────────┬─────────────┬─────────────┬─────────────┤                  │
│  A1         │  A2         │  A3         │  A4         │ ← 第一行：业务概况 │
│  业务概况    │  量子密钥    │  量子隧道    │  加密算法    │   + 量子安全      │
│             │  健康度      │  覆盖率      │  合规分布    │                  │
├─────────────┼─────────────┴─────────────┼─────────────┤                  │
│  B1         │  B2-B3(横跨2格)           │  B4         │ ← 第二行：地图    │
│  量子服务器  │  站点地理分布地图           │  站点/设备   │   + 拓扑         │

├─────────────┼─────────────┬─────────────┼─────────────┤                  │
│  C1         │  C2         │  C3         │  C4         │ ← 第三行：业务性能│
│  性能概况    │  站点时延    │  站点丢包    │  隧道带宽    │                  │

├─────────────┼─────────────┬─────────────┼─────────────┤                  │
```

## 二、各模块详细规格

### 第一行：业务概况 + 量子安全(客户最关心)

#### A1 - 业务概况

| 项 | 说明 |
|---|------|
| 呈现内容 | 站点总数(个)、CPE设备数(个)、隧道总数(条)、业务可用率(%) |
| 呈现形式 | 4个数字指标卡横排(带图标)，底部一条业务可用率进度条(沿用专线管家"99.82%"样式) |
| 客户价值 | 一眼看到组网规模与整体健康度 |
| 数据来源 | SD-WAN手册 1.1工作台(设备总数/隧道数)；SD-WAN手册 2.1网元监控(在线/离线)；业务可用率口径待与厂家对齐 |

#### A2 - 量子密钥健康度

| 项 | 说明 |
|---|------|
| 呈现内容 | 已充注密钥总量(KB)、当前剩余密钥(KB)、密钥剩余比例(%)、最近一次更新时间 |
| 呈现形式 | 环形进度图(剩余/总量比例，<20%变红) + 右侧2个辅助数字 |
| 客户价值 | 量子密钥是"弹药"，客户需要知道弹药还够不够、什么时候补充 |
| 数据来源 | SDWAN量子配置说明 4.2.1(查看秘钥信息：剩余量30624Byte/总量30720Byte/有效期)；量子功能展示P1(密钥信息截图) |

#### A3 - 量子隧道覆盖率

| 项 | 说明 |
|---|------|
| 呈现内容 | 量子加密隧道数、普通IPSec隧道数、量子覆盖率(%) |
| 呈现形式 | 环形图(量子蓝 vs 普通灰)，中间大数字显示量子覆盖率 |
| 客户价值 | 直观回答"我的网络有多少比例受到量子保护" |
| 数据来源 | 量子功能展示P1(隧道列表"量子状态"列：UP表示量子加密、"--"表示无量子)；SDWAN量子配置说明 4.1(量子隧道叠加国密/商密表格) |

#### A4 - 加密算法合规分布

| 项 | 说明 |
|---|------|
| 呈现内容 | 国密(SM3+SM4)隧道数、国际(SHA256+AES256)隧道数、各占比 |
| 呈现形式 | 双色水平堆叠条(国密用红色调、国际用蓝色调) + 百分比标签 |
| 客户价值 | 等保2.0/密评场景下，客户(尤其金融、政务)需要证明"我的加密用的是国密算法" |
| 数据来源 | SDWAN量子配置说明 4.1(隧道表格Hash算法列：SM3/SHA256、加密算法列：SM4/AES256) |

## 四、一屏信息密度校验

对照现有四类管家截图验证信息量是否合理：

| 对比维度 | 专线管家-农业银行 | 本方案(量子+SD-WAN) | 差异说明 |
|---------|------------------|---------------------|---------|
| 第一行 | 内网资源概览(站点/设备/端口/故障时长/可用率) | 业务概况 + 量子密钥 + 量子隧道 + 加密算法 | 用"量子三连"(密钥/覆盖率/算法)替换传统资源概览，强化差异 |
| 第二行 | 站点时延 + 站点抖动 + 站点流速 | 量子服务器 + 地图(2格) + TOP排行 | 地图放大到2格，性能趋势下移到第三行 |
| 第三行 | 设备分布地图 + 短信TOP5 + 故障高频 + 故障原因 | 性能KPI + 时延趋势 + 丢包抖动 + 带宽TOP | 性能集中在第三行，叙事更聚焦 |
| 第四行 | 设备带宽/内存/CPU利用率(3格) | 告警概况 + 等级 + 原因 + 趋势 | 告警集中在第四行 |
